# Capture Issues

If Rockxy shows no useful traffic, the problem is usually one of four things:

- Rockxy did not start cleanly,
- the client is not using Rockxy,
- HTTPS trust is incomplete,
- or the target runtime has its own setup requirements.

## Symptom: nothing appears at all

Check:

- Rockxy is started,
- the proxy port is correct,
- the target client is actually routed through Rockxy.

## Symptom: plain HTTP appears, HTTPS does not

Go straight to [Certificates and Trust](/getting-started/certificates-and-trust) or [Certificate and HTTPS Issues](/troubleshooting/certificates-and-https-issues).

## Symptom: one tool works, another does not

That is often expected. Different tools have different proxy and trust behavior.

Use the appropriate setup page:

- [Browsers and API Clients](/setup-guides/browsers-and-api-clients)
- [Terminal Runtimes and Frameworks](/setup-guides/terminal-runtimes-and-frameworks)
- [Device Setup Issues](/troubleshooting/device-setup-issues)

## Symptom: a local runtime does not emit traffic

The process may ignore the system proxy or require runtime-specific configuration.
