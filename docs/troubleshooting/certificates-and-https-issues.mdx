# Certificate and HTTPS Issues

Most HTTPS failures reduce to trust, SSL proxying scope, or target-client constraints.

## Symptom: CONNECT requests only

The traffic reached Rockxy, but it was not decrypted for inspection. Re-check trust and SSL proxying.

## Symptom: the app says the certificate is invalid

The client does not trust the Rockxy root CA in the path it is using.

## Symptom: one app trusts it, another still fails

That usually means the second app has:

- a separate trust store,
- certificate pinning,
- or a different network stack.
