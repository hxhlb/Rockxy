# Android Device and Emulator Setup

Android setup usually takes more explicit work than macOS clients because proxy configuration, user CA trust, and debug-build behavior all matter separately.

## Physical Android device

Expect to configure:

- device proxy settings,
- certificate installation as a user CA,
- and an app build that actually trusts user-installed certificates when needed.

## Android Emulator

The emulator is often easier to iterate on because you can reset and relaunch faster, but it still needs:

- the right proxy target,
- certificate trust,
- and a test app that follows the intended network stack.

## Why Android debugging can feel inconsistent

Different apps may:

- trust user CAs differently,
- bundle their own networking layers,
- pin certificates,
- or bypass the proxy path entirely.

That is why it helps to start with one known HTTPS probe before testing the real app behavior.
