# Quick Start

This guide takes you from installation to your first useful debugging session. If your goal is “show me the API call, headers, body, and timing right now,” start here.

## Before you start

- Rockxy is running on your Mac.
- You have trusted the Rockxy root certificate.
- The target app or device is configured to use Rockxy as its HTTP/HTTPS proxy, or it follows the macOS system proxy.

If you still need installation details, use [Installation](/installation). If HTTPS requests fail with certificate errors, use [Certificates and Trust](/getting-started/certificates-and-trust).

## 1. Start capture

Click **Start** in the toolbar. Rockxy starts its local proxy and begins recording traffic that is routed through it.

What happens next depends on the client:

- Many macOS apps and browsers will start appearing immediately because they follow the system proxy.
- Some tools need manual proxy and certificate configuration.
- Device and simulator traffic always needs an explicit setup step.

If you are not sure which path applies, open the [Developer Setup Hub](/getting-started/developer-setup-hub) and choose the runtime, browser, framework, or device you are debugging.

## 2. Generate traffic on purpose

Create one request you can easily recognize:

- open a page in a browser,
- trigger one call in your app,
- send a request from Postman or cURL,
- or run the validation request for a target inside the Developer Setup Hub.

<Tip>Press **Cmd+Shift+R** to start capture and **Cmd+.** to stop it.</Tip>

## 3. Inspect the request

Click any request in the list to open the inspector panel. The inspector has several tabs:

- **Headers** for request and response metadata
- **Query** for URL parameters
- **Body** for payload inspection
- **Cookies** and **Set-Cookie** for session work
- **Timing** for latency breakdown
- **Raw** when you need the exact wire representation

If the request was GraphQL-over-HTTP or a WebSocket upgrade, Rockxy exposes protocol-specific inspection automatically. See [GraphQL](/features/graphql) and [WebSocket](/features/websocket).

## 4. Narrow the view

Use the built-in filters to answer a specific question fast:

- filter by protocol,
- search by host, path, method, or status code,
- click a sidebar domain or app scope,
- or split work into workspace tabs when you want separate investigations open at once.

See [Filters and Search](/core-features/filters-and-search) for the full filtering model.

## 5. Replay the call

When you have found the request you care about:

- use **Repeat** for a fast resend,
- use **Edit and Repeat** if you want to change the URL, method, headers, or body first,
- or copy the request as cURL if you want to reproduce it in a terminal.

See [Request Replay](/features/request-replay).

## Common first-run problems

### I only see HTTP, not HTTPS

Your certificate is probably not trusted yet. Use [Certificates and Trust](/getting-started/certificates-and-trust).

### I see no traffic from my tool

The client may ignore the system proxy or use its own certificate store. Start with [Developer Setup Hub](/getting-started/developer-setup-hub) or the [Setup Guides](/setup-guides/browsers-and-api-clients).

### I need to inspect simulator or device traffic

Use:

- [iOS Simulator](/setup-guides/ios-simulator)
- [iOS Device](/setup-guides/ios-device)
- [Android Device and Emulator](/setup-guides/android-device-and-emulator)
